title: Teach Your AI to Code-Switch&#58; How LLMs Are Learning the Secret Dialects
  of Search Engines

Retrieval-Augmented Generation (RAG) has revolutionized how artificial intelligence accesses fresh information. By querying external databases, large language models (LLMs) can answer complex questions without hallucinating. However, developers have long overlooked a hidden bottleneck: not all search engines "speak" the same language.

Treating search as a one-size-fits-all tool, current AI systems often send the exact same query to different search engines. Now, a pioneering study by researchers from institutions including Fudan University, Alibaba DAMO Academy, and Stanford University reveals that LLMs can—and should—be trained to dynamically adapt their search strategies using reinforcement learning (RL).

The core of the issue is what researchers call "structural drift." When an AI fails to find the right document, it is rarely because it misunderstood the user's intent. Instead, it is because it used the wrong style for the specific search algorithm in play. 

To build an intuition, consider the classic, keyword-based search engine **BM25**. If a user asks, *"Who does Dwyane Wade play for right now?"*, an LLM optimized for BM25 learns to strip away natural language and output a concise, keyword-focused query: `"Dwyane Wade current team"`. BM25 thrives on these precise, exact-match targets. 

In contrast, neural search engines like **Contriever** behave entirely differently. Contriever was trained on raw passages of Wikipedia, meaning it prefers queries that look like actual encyclopedia entries. For the question, *"Who wrote God's Plan?"*, a trained LLM generates a complete, hypothetical document-style biography of the rapper Drake, knowing this paragraph-like structure is what Contriever needs to find the actual source. 

Using a reinforcement learning technique called Group Relative Policy Optimization (GRPO), the researchers trained LLMs to automatically discover these optimal "dialects." 

Surprisingly, when the researchers scaled up the query-rewriting LLM to a larger 14-billion parameter model, it began discovering highly effective, non-obvious strategies that even human experts had not anticipated. 

For example, when asked, *"Is it possible to be white and Latino?"*, a human might prompt the system to generate a long, descriptive passage for Contriever. Instead, the 14-billion parameter AI discovered a concise, "statement-style" strategy, rewriting the query simply as: `Being white and Latino`. This seemingly basic phrase yielded a near-perfect retrieval score, vastly outperforming traditional question-style queries. 

The researchers also solved a notorious headache in multi-turn search: training instability. When an AI has to search, evaluate the results, and then search again, reinforcement learning often collapses because it is hard to assign credit to which search step actually helped. The team introduced a "Branching Rollout" strategy. Instead of a single linear path, the system tries multiple search paths at each turn, averaging the results to isolate exactly which query rewrite made the difference.

This research marks a shift from generic AI agents to "environment-aware" systems. By teaching LLMs to code-switch between search engines, future AI assistants will search less, find more, and deliver far more accurate answers.
